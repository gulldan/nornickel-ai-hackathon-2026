# eval-service

`eval-service` считает метрики качества RAG и Фабрики гипотез.

В поставочном `.env.example` плановый запуск выключен:

```text
EVAL_INTERVAL_SEC=0
```

Администратор может запустить оценку из страницы `/admin/metrics`.

## Как работает

1. Сервис берет размеченный набор вопросов.
2. Отправляет вопросы в живой RAG API.
3. Сравнивает источники и ответы с ожиданиями.
4. Сохраняет последнюю сводку качества.
5. Публикует результат в админке.

Локальный запуск:

```bash
python3 evaluate.py
python3 evaluate.py --gate
```

Команды нужно запускать из папки `ml-services/eval-service`.

## Gate

Обычный запуск печатает результат и завершается с кодом 0.

`--gate` включает строгий режим:

1. Код 0: проверка пройдена.
2. Код 1: качество ниже порога.
3. Код 2: ошибка стенда или набора данных.

## Что измеряется

## Поиск

1. `hit@k`: найден ли нужный документ в верхних источниках.
2. `mrr`: насколько высоко первый правильный источник.
3. `ndcg@k`: качество порядка источников.

## Ответ

1. `faithfulness`: ответ опирается на найденные источники.
2. `answer_relevance`: ответ отвечает на вопрос.
3. `context_relevance`: найденный контекст подходит к вопросу.
4. `answer_correctness`: ответ совпадает с эталоном.
5. `answer_similarity`: смысловая близость к эталону.

## Работа стенда

1. Задержка p50.
2. Задержка p95.
3. Средняя задержка.

## Наборы данных

1. `dataset.jsonl`: вопросы для RAG.
2. `dataset-hypotheses.jsonl`: KPI и ожидания для гипотез.

Коммитить можно только синтетические или очищенные наборы. Не добавляйте
клиентские названия, ключи, телефоны, внутренние адреса и реальные секреты.

## Оценка гипотез

Включается переменной:

```text
EVAL_HYPOTHESES=1
```

Метрики:

1. Полнота схемы.
2. Наличие контрдоказательств.
3. Отделение хороших гипотез от слабых.

Эта оценка добавляется к результату и не влияет на основной gate.

## Оценка discovery

Включается переменной:

```text
EVAL_DISCOVERY=1
```

Метрики:

1. Проверяемость.
2. Привязка к источникам.
3. Связь между разными темами.
4. Доля контрдоказательств.
5. Новизна по корпусу.
6. Вклад discovery по сравнению с кластерными гипотезами.

Эта оценка тоже не влияет на основной gate.
