# raptor-worker

`raptor-worker` строит обзорные узлы над обычными фрагментами.

Обзорный узел помогает широким вопросам. При ответе пользователь все равно видит
реальные исходные фрагменты.

## Режим по умолчанию

По умолчанию используется:

```text
RAPTOR_MODE=incremental
```

В этом режиме каждый новый или измененный документ получает стабильный краткий
обзор.
Глобальный граф по всем фрагментам не строится.

## Глобальный режим

Если задать другой `RAPTOR_MODE`, воркер может вызвать `graph-compute` с
гранулярностью `chunk`.

Этот режим полезен для офлайн-экспериментов. Он не является быстрым путем для
демо на большом корпусе.

## Как работает incremental

1. Воркер следит за версией корпуса в Valkey.
2. После паузы получает список документов.
3. Пропускает документы без изменений.
4. Берет ограниченную выборку фрагментов документа.
5. LLM пишет краткий фактический обзор.
6. `f2llm-service` строит смысловой вектор обзора.
7. Воркер пишет точку `node_type=raptor_summary` в Qdrant.
8. Идентификатор точки стабилен, поэтому повторный запуск обновляет запись, а не
   создает дубль.

Если `VLLM_URL` пустой, воркер не строит краткие обзоры.

## Что хранится в Qdrant

1. `node_type=raptor_summary`.
2. `node_level`.
3. `member_ids`: исходные фрагменты.
4. `members`: денормализованные фрагменты для цитирования.

## Основные настройки

1. `RAPTOR_MODE`: режим работы.
2. `RAPTOR_INCREMENTAL_BATCH`: документов за проход.
3. `RAPTOR_INCREMENTAL_WORKERS`: параллельные строители обзоров.
4. `RAPTOR_DOC_MAX_CHUNKS`: максимум фрагментов на обзор документа.
5. `RAPTOR_SUMMARY_MAX_TOKENS`: лимит длины обзора.
6. `RAPTOR_LAST_KEY`: последняя обработанная версия.
7. `GRAPH_COMPUTE_ADDR`: нужен для глобального режима.
8. `EMBEDDINGS_URL` и `EMBEDDINGS_MODEL`: смысловой вектор обзора.
9. `VLLM_URL`, `VLLM_MODEL`, `VLLM_API_KEY`: генератор обзора.

## Проверка

```bash
docker compose -f infra/docker-compose.yml exec -T raptor-worker python worker.py --selfcheck
docker compose -f infra/docker-compose.yml logs --tail=120 raptor-worker
```

Ожидаемое состояние:

1. `rag:raptor:last_epoch` совпадает с `rag:corpus_epoch:shared`.
2. В Qdrant есть точки `node_type=raptor_summary`.
3. Ответы RAG цитируют реальные фрагменты, а не краткий обзор.

Исходный код: `ml-services/raptor-worker`.
